Field Marketing Impact

Know whether the events you funded generated real pipeline and revenue, or just vanity leads that never converted.

Overview

This Skill builds a dashboard grounded in your connected GTM data, with 4 widgets that answer the questions below. Build time on typical data: ~8 min.

What you'll get

Sage assembles a dashboard in roughly ~8 min covering 4 distinct widgets. Every value is source-linked back to 2 integrations and computed using your team's canonical definitions.

Questions answered

This Skill is designed to answer 4 concrete questions GTM teams ask repeatedly:

  • Which events drove qualified pipeline this quarter?
  • What was the cost-per-pipeline-dollar by event tier?
  • Did pipeline from events convert at the same rate as inbound?
  • Which event types are repeat performers vs one-hit wonders?

Before we run

Sage proposes a complete plan and asks you to confirm 3 inputs before any query executes. Defaults are applied if you accept the plan as-is — you can override any of them or set the override as your new canonical.

How it works

A 4-step flow from your connected data to a verified, source-linked output.

01

Connect

Sage reads from your existing source systems — Salesforce, HubSpot. No ETL, schema changes, or warehouse builds required. The data stays where it lives; Sage authenticates and queries it in place.

02

Confirm

Sage proposes a plan and surfaces 3 inputs for you to confirm (listed under "Inputs required" below). You approve or modify the plan. Nothing executes until you sign off.

03

Compute

Sage runs the SQL queries, joins, and Python transforms, applying your team's canonical KDs (Key Definitions) — your ROAS formula, your attribution model, your stage definitions. Every metric uses your math, not a textbook default.

04

Render

Output is generated as a dashboard with 4 widgets (listed under "Outputs generated" below). Every number is source-linked — click any value to see the SQL issued, the rows that contributed, and the formula applied.

Inputs required

The 3 parameters Sage will confirm before running:

  • 1.Which event source — Splash, Bizzabo, manual list, all of them?
  • 2.Define an "event-attributed" opp — first-touch only, or any-touch?
  • 3.Time window — last quarter, year-to-date, or custom?
